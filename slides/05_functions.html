Another way to repeat ourselves: functions!

R is a functional programming language, so we largely prefer functions as the way to do things.

calculate_sample_mean <- function(sample_size) {
  sample_mean <- mean(rnorm(sample_size))
  
  sample_mean
}

calculate_sample_mean(1)
## [1] 0.7548816

Lets break it down

Functions

The general structure of a function is as follows:

function_name <- function(arguments) {
  do_this(arguments)
}

Functions

There are three main parts to a function:

  1. Function name
  2. Function arguments
  3. Function body
function_name <- function(arguments) {
  do_this(arguments)
}

Simple example: printing output

print("Number 1!")
## [1] "Number 1!"
print("Number 2!")
## [1] "Number 2!"
print("Number 3!")
## [1] "Number 3!"
# and so on...
print("Number 10!")
## [1] "Number 10!"

Set up the body of the function:

name <- function() {

Simple example: printing output

Come up with a short, descriptive name:

shout_number <- function() {
  
}

We recommend verb_thing(), so calculate_size() or compare_prices() as vs. fun1(), calc().

Simple example: printing output

Decide what the input to the function will be:

shout_number <- function(number) {
  
}

Simple example: printing output

Do something to that input:

shout_number <- function(number) {
  paste0("Number ", number, "!")
}

Note that we dont need to explicitly call return() - the last line of the code will be the value returned by the function.

Function defaults

We can set values to be defaults, but the user can change them.

Our function from before:

calculate_sample_mean <- function(sample_size) {
  sample_mean <- mean(rnorm(sample_size))
  
  sample_mean
}

Function defaults

We can set values to be defaults, but the user can change them.

We add more arguments, and set default values for them:

calculate_sample_mean <- function(sample_size, 
                                  sample_mean = 0, 
                                  sample_sd = 1) {
  sample_mean <- mean(rnorm(sample_size, 
                            mean = sample_mean, 
                            sd = sample_sd))
  
  sample_mean
}

Function defaults

calculate_sample_mean(10, sample_mean = 6, sample_sd = 2)
## [1] 5.628718
calculate_sample_mean(sample_sd = 2, sample_size = 9)
## [1] 0.5484862

This wont work:

calculate_sample_mean(sample_mean = 5)
Error in rnorm(sample_size, mean = sample_mean, sd = sample_sd) : 
  argument "sample_size" is missing, with no default

Functions in functions

We can pass functions as arguments to other functions. Before:

calculate_sample_mean <- function(sample_size, 
                                  sample_mean = 0, 
                                  sample_sd = 1) {
  sample_mean <- mean(rnorm(sample_size, 
                            mean = sample_mean, 
                            sd = sample_sd))
  
  sample_mean
}

Functions in functions

We can pass functions as arguments to other functions. After:

calculate_sample_mean <- function(sample_size, 
                                  sample_mean = 0, 
                                  sample_sd = 1,
                                  dist_type = rnorm) {
  sample_mean <- mean(dist_type(sample_size, 
                            mean = sample_mean, 
                            sd = sample_sd))
  
  sample_mean
}

Functions in functions

calculate_sample_mean(sample_size = 10, 
                      sample_mean = 0, 
                      sample_sd = 1)
## [1] -0.3279028
calculate_sample_mean(sample_size = 10, 
                      sample_mean = 0, 
                      sample_sd = 1, 
                      dist_type = pnorm)
## [1] 1

calculate_sample_mean() is now probably the wrong name for this function - we should call it calculate_distribution_mean() or something like that.

Detour: probability distributions

Many distributions in R have multiple variants (i.e.the normal distribution has 4 families of functions):

  • dnorm()
  • pnorm()
  • qnorm()
  • rnorm()

These correspond to the density function of the distribution, the cumulative distribution function, the quantile function, and a random sampling function.

Theres also dbinom(), pbinom(), qbinom(), rbinom(), etc.

Read this tutorial for more information.

Best practices for functions

  1. Keep your functions simple and short
  2. Name functions well (verb_thing() is generally good), and document what they take as input and output
  3. Check and debug your functions as you write them, not at the very end
